Different Flavors of Linux

There are many different types or flavors of Linux. In fact there are so many different flavors of Linux that I wouldn’t be able to list them all here. We will just go over the most prominent. There are thousands of distros all based on these main braches of Linux. But a thing to remember is they all use the Linux kernel. There are five main flavors of Linux. Below is a list of these main braches as well as their package managers.

Flavors:

  • Debian (apt)
  • Fedora (YUM but now dnf)
  • Archlinux (pacman)
  • Slackware (pkgtool)
  • Gentoo (portage)

Debian


Debian has it’s start in 1993 and was developed by Ian Murdock. Debian is a stable operating system that is composed of all free software. Debian is also the base of many other operating systems such as Ubuntu, KNOPPIX, and many more. The package management system (Kind of like an apps store) is apt. It also runs .deb executables kind of like windows running .exe files.

Fedora


Fedora was started in 2003, and is supported by Red Had Interprise. As with all of the main branches Fedora is the base of many other operating sytems. Just to name a few Chapeau, Hanthana, and Arquetype just to name a few. Feodra uses rpm software packaging with YUM(But recently changed to dnf) as it’s package manager. And just a fun fact, Fedora is the preferred distro for Linus Torvolds who created the Linux kernel.

Archlinux


Archlinux is a rather unique disro. It is pretty much barebones as in that it doesn’t offer a desktop environment out of the box. It also doesn’t include an installer. Because of this, arch linux is geared toward intermediat to advanced users. Also because of this Archlinux is extremely customizable without having to do really complicated stuff that could break your system. Archlinux was started in 2002 by Judd Vinet. Arch follows a rolling release model meaning that Arch is always supplied with the most bleeding edge software, this can also be a negative beacause this can introduce some unstable software, which could break the system. So you would want to make sure you know what you are doing. Some arch based distros are Antergos, ArchBang, and many others.

Slackware


Slackware Linux is another distro focused on simplicity. Slackware was developed by Patrick Volkerding back in 1993. Slackware is based on Softlanding Linux System. Slackware is also the base for openSUSE, and others. The main package manager for Slack is pkgtool.

Gentoo


Gentoo is probably the most customzable distro outhere short of building your own Linux system. On Gentoo the user must compile the system and the programs that the user wants with the system. Gentoo uses the Portage package manager, and the user compiles and configures their system based on user preference. Gentoo was developed by Daniel Robbins) in 1999. Gentoo is geared toward advanced users.

There are many other main braches of Linux out there but the main ones are noted here. Linux is a huge tree of main branches with new operating systems based off of them and newer systems based off of those systems. Linux is fast growing and ever expanding, and will only grow larger. But knowing the roots of the modern Linux operating sytems is pretty important. Trying out these base systems is a great learning experience. I encourage you to go out and try using one of these systems as your main OS.
